# Comparison Page (`templates/comparison.html`)

## Purpose
Interactive dashboard to compare model metrics, show generated visualizations, and inspect runtime/system metrics.

## Structure
- **Tabs**: Overview, Precision & Recall, Distributions, Embeddings, Correlations, Runtime Metrics, 3D View.
- **Overview (varied charts)**: GC Error (column), K-mer JS (area), Uniqueness (horizontal bar), Edit Distance (line), Motif Score (area), Radar (overall performance).
- **Precision & Recall**: Two spline charts showing reliability and coverage.
- **Distributions**: GC content distribution and GC% boxplot.
- **Embeddings**: 2D PCA scatter of k-mer vectors per model.
- **Correlations**: Heatmaps for 2-mer / 3-mer correlations across models.
- **Runtime Metrics**: Static PNGs (epoch time, CPU/RAM, GPU mem, gen time) produced by metrics scripts.
- **3D View**: 3D loss landscape and 3D GPU vs time scatter (PNG).

## Data Inputs
- `/metrics_json` for core metrics (GC_error, kmer_JS, Uniqueness, EditDist, MotifScore, GC_content, Precision, Recall).
- Static images from `static/metrics/*.png` generated by metrics scripts.

## Key JS
- `loadMetrics()` builds charts via Highcharts, initializes tabs, and sets color map.
- Tab controller toggles `.active` class on buttons/panels.
- Correlation heatmaps fetch `/correlation_heatmap_data` with k selection.

## Rationale
- Tabbed layout shortens page length while keeping all visualizations.
- Mixed chart types reduce visual monotony.
- Static PNGs for runtime metrics keep load light; generation is offline via scripts.
